# v1.0.22

이번 릴리스는 RouteFlow를 운영 환경에 올렸을 때 체감되는 병목과 운영 기능을 다듬는 데 집중했습니다.

## 주요 변경

- 엔진 구독 라우팅을 전체 구독자 순회 방식에서 endpoint별 역방향 인덱스 방식으로 바꿨습니다.
- SQLite `RouteStore`에 테이블별 64개 LRU statement cache를 추가했습니다.
- `SIGTERM`, `SIGINT` 수신 시 10초 drain 후 `close()`를 호출하는 graceful shutdown을 기본 지원합니다.
- `GET /_health` 엔드포인트를 자동 등록하고, 모든 HTTP 요청에 `X-Request-ID`를 자동 생성·전달합니다.

## 성능 최적화

### 1. 엔진 구독 라우팅: O(N) → O(matching)

이전에는 DB 변경 이벤트가 들어올 때마다 모든 구독자를 순회하며 route pattern regex를 검사했습니다.

- 이전: 구독자 50,000명이라면 이벤트마다 최대 50,000번 검사
- 이후: `subscriptionsByEndpoint` 역방향 인덱스에서 해당 endpoint/path 그룹만 조회

즉, 채팅방·룸·테넌트처럼 concrete path가 명확한 앱에서는 "전체 구독자 수"가 아니라 "실제로 영향을 받는 구독자 수"에 비례해 fan-out 비용이 결정됩니다.

## SQLite 최적화

`RouteStore`는 `db.prepare(sql)` 결과를 테이블별 LRU 캐시에 유지합니다.

- 캐시 크기: 테이블당 64개 statement
- 대상: `SELECT`, `INSERT`, `UPDATE`, `DELETE` 등 반복되는 CRUD 쿼리
- 효과: parse/compile 비용을 줄여 일반적인 CRUD 레이턴시를 낮춤

SQLite 내장 statement cache만으로는 테이블 수가 많을 때 eviction이 잦았는데, RouteFlow 쪽 캐시가 이를 보완합니다.

또한 SQLite 엔트리포인트는 ESM `import`뿐 아니라 CommonJS `require('routeflow-api/sqlite')`에서도 안전하게 로드되도록 정리했습니다.

## 운영 기능

### Graceful shutdown

컨테이너 환경에서 프로세스가 종료될 때 다음 흐름으로 정리됩니다.

1. `SIGTERM` 또는 `SIGINT` 수신
2. 최대 10초 동안 in-flight 요청 drain
3. `app.close()` 실행
4. 정상 종료 시 `process.exit(0)`

이제 Kubernetes, Docker, Fly.io 같은 환경에서 종료 중 요청 끊김과 DB 연결 누수 가능성을 줄일 수 있습니다.

### `/_health`

별도 설정 없이 `GET /_health`가 등록됩니다.

```json
{
  "status": "ok",
  "uptime": 123,
  "timestamp": "2026-04-17T00:00:00.000Z"
}
```

Kubernetes liveness probe, 로드밸런서 상태 확인, 기본 헬스체크에 바로 사용할 수 있습니다.

### `X-Request-ID`

모든 HTTP 요청은 다음 동작을 기본으로 가집니다.

- 요청에 `X-Request-ID`가 있으면 그대로 전달
- 없으면 서버가 자동 생성
- 응답 헤더에도 동일한 `X-Request-ID`를 포함
- 핸들러와 미들웨어에서는 `ctx.requestId`로 접근 가능

```ts
app.use(async (ctx, next) => {
  console.log(`[${ctx.requestId}] ${ctx.headers['x-forwarded-for'] ?? 'unknown'}`)
  await next()
})
```

## 대상 사용자

이번 릴리스는 특히 아래 환경에서 효과가 큽니다.

- 룸/채널/테넌트 단위 구독이 많은 실시간 앱
- SQLite 기반 로컬 퍼시스턴스 또는 단일 노드 배포
- Kubernetes, Docker 같은 오케스트레이션 환경
- API gateway, reverse proxy, APM과 함께 request tracing이 필요한 운영 환경
