# RouteFlow v1.0.24

## Highlights

- Query-aware live fan-out is now stable for default reactive routes even when
  query parameter order differs.
- `relations.<name>.watch` can be declared explicitly for `liveInclude`
  relation recomputation when the alias and backing change source differ.
- Input sanitization now strips dangerous prototype-pollution keys from plain
  object bodies and string-record request sources.
- `query: 'auto'` now validates pagination and ordering inputs before forwarding
  them to stores.
- Fastify moved to `5.8.5` and the Snowflake peer minimum moved to `2.0.4`.

## Validation

- `pnpm --filter routeflow-api test`
- `pnpm --filter routeflow-api build`
- `pnpm run example:differentiation:smoke`
- `pnpm run test:e2e`
